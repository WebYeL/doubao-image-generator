# 部署说明

## 环境要求

### 后端
- Python 3.8+
- pip
- ARK_API_KEY（火山引擎API密钥）

### 前端
- Node.js 16+
- npm 或 yarn

---

## 开发环境部署

### 方式一：使用启动脚本

#### Windows
```bash
# 启动后端
cd scripts
start_backend.bat

# 新开终端，启动前端
start_frontend.bat
```

#### Linux/Mac
```bash
chmod +x scripts/start_all.sh
./scripts/start_all.sh
```

### 方式二：手动启动

#### 1. 启动后端

```bash
cd backend

# 创建虚拟环境（推荐）
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或 venv\Scripts\activate  # Windows

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入 ARK_API_KEY

# 启动服务
uvicorn app.main:app --host 0.0.0.0 --port 8886 --reload
```

#### 2. 启动前端

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

访问：
- 前端：http://localhost:5173
- 后端API：http://localhost:8886
- API文档：http://localhost:8886/docs

## 生产环境部署

### 1. 后端部署

#### 使用Gunicorn

```bash
cd backend
pip install gunicorn
gunicorn -w 4 -k uvicorn.workers.UvicornWorker app.main:app --bind 0.0.0.0:8886
```

#### 使用Docker

创建 `Dockerfile`：

```dockerfile
FROM python:3.10-slim

WORKDIR /app

COPY backend/requirements.txt ./
RUN pip install --no-cache-dir -r requirements.txt

COPY backend/ ./

# 创建存储目录
RUN mkdir -p generated_images generated_videos

EXPOSE 8886

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8886"]
```

构建和运行：
```bash
docker build -t doubao-backend .
docker run -d -p 8886:8886 \
  --env-file backend/.env \
  doubao-backend
```

#### 使用Nginx反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8886;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /api {
        proxy_pass http://127.0.0.1:8886;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 2. 前端部署

#### 构建生产版本

```bash
cd frontend
npm install
npm run build
```

构建产物在 `dist` 目录。

#### 部署选项

**选项A：Nginx静态文件服务**

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://127.0.0.1:8886;
    }
}
```

**选项B：部署到子路径**

修改 `vite.config.js`：

```javascript
export default defineConfig({
  base: '/image-generator/',
  // ...
})
```

修改Nginx配置：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location /image-generator/ {
        alias /path/to/frontend/dist/;
        try_files $uri $uri/ /image-generator/index.html;
    }

    location /api {
        proxy_pass http://127.0.0.1:8886;
    }
}
```

### 3. 使用Systemd服务（Linux）

创建服务文件 `/etc/systemd/system/doubao-backend.service`：

```ini
[Unit]
Description=Doubao Image/Video Generation Backend
After=network.target

[Service]
User=www-data
WorkingDirectory=/path/to/doubao_test/backend
Environment="PATH=/path/to/venv/bin"
EnvironmentFile=/path/to/doubao_test/backend/.env
ExecStart=/path/to/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8886
Restart=always

[Install]
WantedBy=multi-user.target
```

启用服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable doubao-backend
sudo systemctl start doubao-backend
sudo systemctl status doubao-backend
```

## 配置说明

### 环境变量

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| ARK_API_KEY | 是 | - | 火山引擎API密钥 |
| ARK_BASE_URL | 否 | https://ark.cn-beijing.volces.com/api/v3 | API地址 |
| ARK_MODEL_NAME | 否 | doubao-seedream-5-0-260128 | 图片模型名称 |
| VIDEO_MODEL_NAME | 否 | doubao-seedance-2-0-pro-260416 | 视频模型名称 |
| HOST | 否 | 0.0.0.0 | 监听地址 |
| PORT | 否 | 8886 | 监听端口 |
| DEBUG | 否 | true | 调试模式 |
| IMAGE_STORAGE_PATH | 否 | ./generated_images | 图片存储路径 |
| VIDEO_STORAGE_PATH | 否 | ./generated_videos | 视频存储路径 |
| VIDEO_POLL_INTERVAL | 否 | 10 | 视频任务轮询间隔（秒） |
| VIDEO_MAX_POLL_TIME | 否 | 600 | 视频最大轮询时间（秒） |
| CORS_ORIGINS | 否 | http://localhost:5173 | 允许的源（逗号分隔） |

### 目录权限

确保存储目录可写：

```bash
mkdir -p generated_images generated_videos
chmod 755 generated_images generated_videos
```

### 清理

清理旧的生成文件：

```python
# backend/scripts/cleanup.py
import os
import time
from pathlib import Path

def cleanup_old_files(directory, days=30):
    """删除指定天数前的文件"""
    threshold = time.time() - (days * 86400)
    for file in Path(directory).glob('*'):
        if file.is_file() and file.stat().st_mtime < threshold:
            file.unlink()
            print(f"Deleted: {file}")

if __name__ == '__main__':
    cleanup_old_files('./generated_images', days=30)
    cleanup_old_files('./generated_videos', days=30)
```

## 故障排查

### 常见问题

**1. API Key无效**
```
错误: API key not configured
解决: 检查 .env 文件中的 ARK_API_KEY
```

**2. 前端无法连接后端**
```
解决: 检查 CORS 配置是否正确
     检查后端服务是否运行在 http://localhost:8886
     检查防火墙设置
```

**3. 视频生成失败**
```
解决: 检查 VIDEO_MODEL_NAME 配置
     确保模型版本可用
     检查网络连接
```

**4. 视频下载失败**
```
解决: 检查存储目录权限
     检查磁盘空间
     确��视频文件已生成
```

**5. 生成超时**
```
解决: 增加请求超时时间
     调整轮询参数（VIDEO_MAX_POLL_TIME）
     检查网络连接
```

## 安全建议

1. **保护API密钥**
   - 不要将 .env 文件提交到版本控制
   - 使用环境变量而不是硬编码密钥
   - 定期轮换API密钥

2. **限制访问**
   - 生产环境使用HTTPS
   - 配置防火墙规则
   - 使用速率限制（可在Nginx中配置）

3. **定期更新**
   - 更新依赖包
   - 更新Python和Node.js版本
   - 关注火山引擎API的版本更新

4. **监控**
   - 设置日志告警
   - 监控资源使用（CPU、内存、磁盘）
   - 监控API调用量和成功率
   - 定期清理历史文件（建议30天）

   ```nginx
   gzip on;
   ```
